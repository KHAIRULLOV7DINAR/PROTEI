Для запуска программы (в каталоге DEVELOP):  
  
mkdir build  
cd build  
cmake ..  [DBUILD_TESTS=ON] //для сборки с тестами  
cmake --build .  
ctest -V                    //запуск тестов  
  
HOMETASK_01:  
1. Для сборки программы используется cmake, CMakeLists.txt находится в корневой папке проекта;  
2. Программа принимает аргументы командной строки вида:  
HOMETASK_01 -a 127.0.0.1 -p 5555 -r Client -i 0 -L mylib  
3. Программа реализует текстовый интерфейс через меню (команды не чувствительны к регистру):  
3.1. Ввод строкового псевдонима программы (строковое значение)     - name;  
3.2. Ввод типа вектора (строковое значение)                        - type;  
3.3. Ввод числового четырёхмерного вектора (int, double, float)    - vector;  
3.4. Выход                                                         - exit;  
3.5. Вывод аргументов командной строки                             - console;  
3.6. Вывод аргументов, получаемых из меню                          - menu;

Критичные ошибки:  
Отсутствие аргументов ip-адреса и названия библиотеки в аргументах консольной команды;  
Ввод невалидных данных ip-адреса (не в формате ipv4: 255.255.255.255);  
Указание флага без значения в командной строке (... -p -i 0 ... указан флаг -p, но не указано значение);  
Указание не предусмотренных флагов для нашей программы (-A, к примеру);  
Двойное указание флагов командной строки (... -p 123 -p 124 ...).  
  
Они все относятся к ошибкам при указании аргументов командной строки, так они указываются до выполнения программы один раз. Но программа не сможет выполняться с некорректными данными, следовательно, необходимо остановить выполнение программы.  
  
Некритичные ошибки:  
Введена некорректная команда при работе с cli-меню;  
Введен некорректный тип вектора в меню;  
Введено некорректное значение вектора в меню;  
Попытка ввести значения вектора до указания типа;  
W-компонента вектора = 0.  
  
Все эти ошибки возникают при работе пользователя с меню, так как в случае указания им некорректных данных, мы можем сообщить ему об ошибки и запросить ввод новых данных, эти ошибки некритичны.
